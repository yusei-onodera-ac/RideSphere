# RideSphere MVP計画

## 最初の完成形

最初の完成形は、要件全体を一度に作り込まず、**地図付き配車管理PWA**に絞ります。

## MVPで提供する業務フロー

1. 管理者が予約を登録する。
2. システムが乗車場所に近い空き運転手を検索する。
3. 距離、到着時間、勤務状態、車両利用可否、対応可能エリアで候補を優先順位付けする。
4. 管理者が候補運転手へ配車通知を送る。
5. 運転手がPWAで通知を確認して承認する。
6. 配車を確定し、運行完了を記録する。

## MVPに含めた画面・機能

- 管理者用の予約登録フォーム
- 管理者用の現在地マップ風ビュー
- 乗車場所、目的地、ルート、運転手現在地、車両状態の可視化
- 近くの空き運転手候補リスト
- 運転手用PWA画面
- 現在地共有ボタン
- 配車通知、承認、運行完了ボタン
- オフライン時の最低限表示に向けたService Workerキャッシュ
- ホーム画面追加に向けたWeb App Manifest

## 自動采配の判定条件

初期実装では、次の条件を満たす運転手だけを候補にします。

- 現在「空き」状態
- 勤務時間内
- 車両が利用可能
- 指名予約専用ではない
- 対応可能エリア内
- 休憩中・退勤済みではない

候補は、距離と到着予定時間をもとにスコアリングして並べます。
